# Profession And Skill Unlocks

## 1. Purpose

This document defines how professions, profession skills, and long-term unlock structure should work in the first production version.

The goal is to make profession choice:

- visible
- mechanically meaningful
- thematically expressive
- still manageable for a small team

## 2. Core profession rule

Professions should be treated as pre-authored progression identities that players can selectively invest into.

Key rule:

- players are not permanently locked into only one abstract fantasy forever
- profession skills are purchased by player choice in the main-god space
- the game should allow directed preference rather than fully random class assignment

This keeps long-term growth flexible while preserving authored identity.

## 3. First unlock moment

The first profession skill purchase is a major identity threshold.

Once the player purchases a profession skill for the first time:

- the profession becomes visibly active
- the player's carry container presentation changes
- the battle space begins to gain profession-dependent identity
- profession-specific presentation begins to appear
- profession-specific rules are allowed to enter the run layer

This moment should feel like a commitment point, not a minor stat purchase.

## 4. Profession presentation layer

Each profession should affect visible presentation in at least these ways:

- carry container form
- battle-space presentation or structure cues
- UI identity cues
- skill icon language
- item affinity cues where appropriate

Example:

- a xianxia-aligned profession should replace a generic backpack with a `spatial ring`

The visual shift should immediately communicate:

- "my character logic has changed"
- "my build space now belongs to a profession fantasy"

## 5. Profession system scope for first version

The first production version should keep profession output compact.

Recommended minimum profession package:

- one identity concept
- one container or bag presentation
- one battle-space variation rule or presentation change
- one core passive rule
- one small skill branch
- one item affinity bias

Do not start with:

- many professions at equal depth
- large talent trees
- profession-exclusive full content packs

## 6. Skill purchase model

Profession skills should be purchased in the main-god space.

Recommended purchase rule:

- players unlock skills by spending the dedicated profession-unlock resource once that resource is formally defined
- skill purchase should be intentional, not random
- the player should always understand what a skill changes before buying it

The first version should favor:

- clear text
- small choice count
- visible payoff

## 7. Profession skill categories

Recommended first-version skill categories:

- battle-space modifier
- storage-space modifier
- shop or economy bias
- item-family affinity
- one profession-specific rule twist

This keeps the system connected to the core loop rather than drifting into abstract RPG stats.

## 8. Connection to the build loop

Profession skills should reinforce the main game loop:

- preparation
- arrangement
- adaptation
- validation

Good profession skills should affect questions like:

- how much active battle space is available
- how the battle space is shaped or structured after profession unlock
- what kind of support pieces become easier to use
- how storage interacts with run planning
- what kinds of rewards or shop options become more appealing

Bad profession skills are skills that:

- only add generic stats
- do not change decisions
- make all builds look the same

## 9. Profession-specific features

Each profession should eventually gain one or more special features.

Recommended first-version rule:

- each profession gets one profession-specific feature that can be explained in one sentence

Examples of acceptable scope:

- a xianxia profession gains better synergy with directional spirit tools
- a scavenger-style profession gains one storage-related conversion trick
- a ritual profession gains one special support-link rule

The first version should prove distinct play feel, not content volume.

## 10. Relationship to item and world design

Profession identity should influence but not fully replace world identity.

Design rule:

- worlds define external conditions
- professions define player-side bias

This distinction is important because:

- world variety creates encounter freshness
- profession identity creates player ownership

## 11. UX requirements

The profession system must make these things obvious:

- which profession identity is currently active
- which skills have been unlocked
- which skills are available next
- what changed after the newest purchase
- how the carry container has changed
- how the battle space changed after profession activation

If players unlock a profession skill and do not notice it during the next run, the system is underperforming.

## 12. Validation questions

This system is working only if:

- the first skill purchase feels memorable
- players can explain what their profession currently does
- the profession changes both presentation and decision-making
- players want to pursue a preferred profession direction
- profession identity does not overwhelm the base build system

## 13. Immediate production implications

The planning set should later include:

- first-version profession roster
- profession-specific container presentation sheet
- skill unlock resource definition
- first profession skill table
- profession-to-item affinity table
- profession UI state specification
