# 首批职业被动与解锁分支

## 1. 文档目的

这份文档用于定义首发职业的第一层可实现规则。

目标是把职业从“身份概念”推进到“可做原型”的程度。

每个首发职业都应先拥有：

- 一个首个核心被动
- 一条小型首批解锁分支
- 一句清晰的玩法承诺

## 2. 设计规则

首批职业规则应满足：

- 容易理解
- 在正常游玩中能被明显感知
- 直接作用于构筑决策
- 范围足够小，便于快速原型验证

应避免首批职业规则出现这些问题：

- 只提供平面的数值加成
- 需要大量职业专属内容才能成立
- 需要很重的教学解释
- 压过核心空间构筑循环本身

## 3. 修士

### 核心玩法承诺

修士应给人一种“精密构筑大师”的感觉，玩家会因为正确的摆放、朝向和灵性支撑而得到更高收益。

### 首个被动：灵脉校准

规则：

- 当具有朝向要求或供能链要求的物品被完整、干净地满足条件时，会获得额外效果收益

玩家感知：

- 正确摆放会被更明显地奖励
- “干净”的构筑会比“勉强能用”的构筑更强

为什么这样合适：

- 强化项目的核心构筑幻想
- 鼓励可读的布局优化
- 也天然贴合修仙向的支撑逻辑

### 首条解锁分支

#### 节点 1：符箓和鸣

效果：

- 当符箓类支撑件相邻于有效目标时，提供更高一点的支撑收益

#### 节点 2：灵石导通

效果：

- 灵石类供能物在激活链路中损失更少效率

#### 节点 3：兵刃识位

效果：

- 朝向正确的武器类物品获得更明确的收益加成

分支目的：

- 强化朝向
- 强化支撑清晰度
- 强化供能身份

## 4. 魔法师

### 核心玩法承诺

魔法师应给人一种“先布置、后爆发”的感觉，通过充能与法术结构把准备转化成清晰的魔法回报。

### 首个被动：蓄能施法

规则：

- 当法术类物品在至少满足一项魔法准备条件后触发时，会获得额外收益

玩家感知：

- 魔法师会因为时序和准备而变强
- 法术爆发应显得是“我布出来的”，而不是随机发生

为什么这样合适：

- 它和修士形成了明显不同的幻想
- 它天然支持可读的法术连锁时刻
- 它会让玩家更在意准备与爆发之间的关系

### 首条解锁分支

#### 节点 1：速构法印

效果：

- 每场战斗中的第一个魔法支撑件更容易或更快进入有效状态

#### 节点 2：储能回路

效果：

- 充能型法术物品在准备阶段到爆发阶段之间保留更多已积累价值

#### 节点 3：奥术回响

效果：

- 每场战斗中，在满足条件时，一次法术触发获得较轻量的额外回响效果

分支目的：

- 强化魔法准备
- 强化充能兑现
- 强化法术身份

## 5. 人工智能

### 核心玩法承诺

人工智能职业应给人一种“设计机器逻辑”的感觉，结构、链路和一致性比单纯的幻想炫酷更重要。

### 首个被动：预测例程

规则：

- 当某个物品处于有效逻辑链或支撑链中，并且其依赖条件被完整满足时，它会获得更高的一致性收益

玩家感知：

- 完整、有组织的系统会比松散拼装的系统更稳定
- 这个职业奖励的是规划，而不只是把强件全塞进去

为什么这样合适：

- 非常贴合这个项目“系统优先”的气质
- 它带来明确的非幻想型身份
- 也天然支持可读的自动化逻辑

### 首条解锁分支

#### 节点 1：高效处理

效果：

- 当逻辑链路成立时，支撑件或逻辑件消耗的运行代价更低

#### 节点 2：冗余保护

效果：

- 每场战斗中，一次失败的结构化激活可以被部分补救

#### 节点 3：优化周期

效果：

- 当 build 保持稳定链接结构时，会在一个周期结束后获得小幅效率奖励

分支目的：

- 强化逻辑链身份
- 减少失败时的挫败尖峰
- 奖励干净的自动化结构

## 6. 共通实现规则

为了让首批职业内容保持可控：

- 每个职业的首条分支只做 3 个早期节点
- 每个节点只改动一个玩家能理解的环节
- 职业被动尽量复用已有道具标签
- 职业规则应能在 UI 中被检查到

## 7. 验证问题

只有当以下问题大多能回答“是”时，这批职业被动才算成立：

- 玩家在一两局内就能感受到差异
- 每个职业都会改变道具价值判断
- 每个职业都会改变玩家的空间摆放方式
- 没有哪个职业需要很重的额外解释才能理解其幻想
- 整个游戏仍像一个统一产品，而不是三个互不相干的小系统

## 8. 对制作的直接影响

下一层规划应继续定义：

- 职业判定所需的精确道具标签
- 每个被动的 UI 文案
- 每个分支节点的解锁顺序与消耗
- 首批职业专属教学提示
