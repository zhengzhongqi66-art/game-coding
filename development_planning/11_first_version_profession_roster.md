# First-Version Profession Roster

## 1. Purpose

This document defines the recommended profession roster for the first production version.

The goal is to choose a profession set that:

- supports the core build loop
- creates distinct play feel
- fits the current xianxia-first direction
- stays achievable for a small team

## 2. Scope rule

The first version should not attempt a large class roster.

Recommended first-version target:

- 2 to 3 professions total
- 1 fully polished lead profession
- 1 secondary profession with clear mechanical difference
- 1 optional reserve concept only if production capacity remains healthy

Reason:

- professions affect UI, economy, space logic, and item affinity
- each additional profession multiplies balancing and presentation cost

## 3. Profession selection criteria

A first-version profession is worth shipping only if it provides:

- a different build-space fantasy
- a different decision bias
- a distinct item affinity
- a visible container identity
- a profession-specific rule that can be explained simply

If two professions differ only by theme words, one should be cut.

## 4. Recommended profession roster

### A. Lead profession: Cultivator

Role:

- the flagship first-version identity
- the profession most aligned with the current xianxia-first direction

Container identity:

- `Spatial Ring`

Core passive direction:

- gains stronger value from directional and energy-linked tools

Profession-specific feature:

- spirit tools and talisman-style support pieces are easier to activate efficiently

Item affinity bias:

- talismans
- spirit stones
- furnaces
- directional weapons

Why it belongs in version one:

- strongest bridge between existing planning and recognizable fantasy
- naturally supports orientation, support, and energy systems
- likely the best visual hook for screenshots and early communication

### B. Secondary profession: Mage

Role:

- a contrast profession focused on spell structure, ranged payoff, and magical control

Container identity:

- `Arcane Grimoire`

Core passive direction:

- gains more value from spell-form items, charged effects, and magical sequencing

Profession-specific feature:

- gains a cleaner payoff path for spell-combo setups and delayed magical bursts

Item affinity bias:

- spell focuses
- charged artifacts
- burst tools
- magical support pieces

Why it belongs in version one:

- creates a clear fantasy contrast to Cultivator
- supports highly readable combat payoff through spell chains and magical bursts
- gives the project a stronger cross-theme appeal beyond xianxia-only framing

### C. Third profession: Artificial Intelligence

Role:

- a systems-driven profession focused on logic, automation, and optimization

Container identity:

- `Data Core`

Core passive direction:

- gains more value from processing links, modular routines, and high-precision optimization

Profession-specific feature:

- gains one profession rule that improves structured automation or predictive activation logic

Item affinity bias:

- processors
- drone-like modules
- logic relays
- optimization tools

Why it belongs in version one:

- gives the roster a strong non-fantasy contrast
- fits the game's system-first identity unusually well
- can create a very different fantasy of "designing a machine mind build"

## 5. Reserve professions

The following professions should be treated as reserve concepts, not first-version priorities:

### Reserve profession: Scavenger

- fantasy: adaptive salvager
- focus: storage and conversion
- container: Modular Field Pack
- use later if storage identity needs stronger emphasis

### Reserve profession: Ritualist

- fantasy: preparation and setup specialist
- focus: links and timing
- container: Seal Case
- use later if support-network gameplay needs deeper specialization

## 6. Recommended launch order

Recommended production order:

1. Cultivator
2. Mage
3. Artificial Intelligence only if capacity allows

Do not split early production across all professions equally.

The lead profession should prove:

- profession presentation value
- container identity value
- profession-specific rule value

before the second profession expands scope.

### Mage

- focus: magical sequencing and charged payoff

### Artificial Intelligence

- Scavenger: storage and conversion identity

## 8. First-version control rules

To keep professions manageable:

- each profession gets one signature passive
- each profession gets one visible container identity
- each profession gets one small unlock branch
- each profession gets one item-affinity statement

Avoid in version one:

- profession-exclusive encounter pools
- profession-exclusive bosses
- large profession-only UI systems
- deep branching profession trees

## 9. Validation questions

This roster is working only if:

- players can immediately tell the difference between professions
- the container identity matters emotionally and visually
- profession choice changes build behavior in a noticeable way
- the second profession adds contrast rather than noise
- the optional third profession can be cut without harming the core demo

## 10. Immediate production implications

The next planning layer should define:

- exact first passive for Cultivator
- exact first passive for Mage
- exact first passive for Artificial Intelligence if shipped in version one
- first unlock branch for each shipped profession
- profession-specific container mockups
- item tagging required for profession affinity
