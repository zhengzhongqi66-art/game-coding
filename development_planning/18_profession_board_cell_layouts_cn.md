# 职业棋盘格位布局

## 1. 文档目的

这份文档用于定义职业化战斗空间变体的第一版特殊格坐标布局。

目标是把职业棋盘具体到足以支持：

- UI 线框
- 棋盘渲染
- 格子高亮
- 规则状态原型

## 2. 坐标约定

战斗空间使用：

- `6 列 x 5 行`
- 列编号：`1` 到 `6`
- 行编号：`1` 到 `5`
- 坐标格式：`(列,行)`

左上角是 `(1,1)`。  
右下角是 `(6,5)`。

## 3. 基础棋盘

基础棋盘本身没有职业特殊格。

只有在通过 DNA 激活职业后，棋盘状态才会变成职业化棋盘。

## 4. 修士棋盘布局

### 变体身份

- `阵图棋盘`

### 特殊格

- 阵心格：`(3,3)` 与 `(4,3)`
- 上阵位格：`(3,2)` 与 `(4,2)`
- 下阵位格：`(3,4)` 与 `(4,4)`

### 布局逻辑

这个布局会形成：

- 中央阵带
- 很强的中盘身份
- 对接近对称或优雅支撑结构的天然支持

### 预期用途

- 阵位附近的结构应更适合承载朝向型物品
- 围绕中央阵带摆放支撑件应显得更有意义

### UI 备注

- 阵心格应比普通阵位格更明显
- 整体要读起来像阵法，不要像强制标准答案模板

## 5. 魔法师棋盘布局

### 变体身份

- `奥术施法棋盘`

### 特殊格

- 左施法通道：`(2,1)`、`(2,2)`、`(2,3)`、`(2,4)`、`(2,5)`
- 右施法通道：`(5,1)`、`(5,2)`、`(5,3)`、`(5,4)`、`(5,5)`
- 奥术锚点格：`(3,3)` 与 `(4,3)`

### 布局逻辑

这个布局会形成：

- 两条可读性很强的纵向施法通道
- 一对用于法术会聚或准备的中央锚点

### 预期用途

- 法术类道具应更像是在施法结构中被引导
- 中央锚点应奖励有计划的魔法爆发

### UI 备注

- 施法通道应表现为“流动路线”，不要像墙或限制
- 棋盘要传达魔力流动，而不是把玩家锁进单一路径

## 6. 人工智能棋盘布局

### 变体身份

- `处理网格`

### 特殊格

- 左处理通道：`(2,2)`、`(2,3)`、`(2,4)`
- 右处理通道：`(5,2)`、`(5,3)`、`(5,4)`
- 上桥接格：`(3,2)` 与 `(4,2)`
- 下桥接格：`(3,4)` 与 `(4,4)`
- 中央处理区：`(3,3)` 与 `(4,3)`

### 布局逻辑

这个布局会形成：

- 两侧处理通道
- 中央结构化处理区
- 外侧与中央之间清晰的桥接逻辑

### 预期用途

- 逻辑链物品在沿着这些结构布线时应显得更顺
- 棋盘整体应传达“计算秩序感”

### UI 备注

- 桥接格应帮助玩家理解连接关系
- 棋盘要显得结构化，但不能视觉噪音过高

## 7. 首版控制规则

这些特殊格应被理解为：

- 激励格
- 身份格
- 解释格

而不应被理解为：

- 大多数情况下的硬限制格
- 每套 build 都必须使用的强制格
- 粗暴的大数值乘区

## 8. 共通实现建议

建议的首版棋盘渲染方式：

- 普通格使用中性样式
- 职业特殊格使用职业主题叠加层
- 最强身份格使用更明显的边框或微发光

这样既能保留可读性，又能让不同职业棋盘有鲜明差异。

## 9. 验证问题

只有当以下问题大多能回答“是”时，这些布局才算成立：

- 玩家能否一眼看出职业棋盘的图案？
- 特殊格激励是否真的改变了摆放决策？
- 棋盘中央区域是否会因职业不同而承担不同意义？
- 棋盘整体是否仍然足够开放，允许玩家实验？

## 10. 对制作的直接影响

下一层规划应继续定义：

- 每种特殊格到底提供什么效果
- 棋盘叠加层的视觉语言
- 特殊格的悬停与 tooltip 行为
