# Content And Pipeline

## 1. Content creation principle

The project should use "small world packages" instead of giant thematic expansions.

For each world, only commit to:

- 3 visual symbols
- 2 enemy behavior ideas
- 1 stage rule
- 1 reward family
- 1 side event hook

This matches the sampling logic from the source planning.

## 2. Content production order

Recommended order for every new world or theme:

1. Define its gameplay rule.
2. Define its reward identity.
3. Define the enemy behavior that expresses the rule.
4. Define visual symbols.
5. Define one boss.
6. Add one optional side event.

This prevents art-first drift.

## 3. Item production pipeline

Each item should go through:

1. Role definition
2. Trigger definition
3. Space requirement definition
4. Support requirement definition
5. Counterplay or weakness definition
6. Readability pass
7. Visual icon pass
8. Balance pass

Simple item design template:

- Name
- Theme family
- Shape and size
- Orientation rules
- Adjacency rules
- Energy use or supply
- Trigger timing
- Main payoff
- Failure case
- Visual cue

## 4. Encounter production pipeline

Each encounter should specify:

- combat role
- enemy mix
- stage modifier if any
- expected danger pattern
- likely counter-builds
- reward logic

Every encounter should test something meaningful, such as:

- front-loaded damage
- sustain check
- energy disruption
- formation pressure
- anti-summon
- anti-greedy layout

## 5. Art direction recommendation

The game should use a modular visual style that can survive theme rotation.

Recommended style principles:

- readable item silhouettes first
- strong icon clarity over detail overload
- battle effects that show cause and effect clearly
- compact world identity through a few repeated motifs
- playful tone through naming and presentation, not text walls

## 6. Audio direction recommendation

Need modular audio layers:

- hub ambience
- world ambience
- item activation cues
- major proc and combo cues
- elite and boss warning cues
- reward opening cues

Audio goal:

- strengthen feedback loops without making battle unreadable

## 7. Team workflow recommendation

For a small team, responsibilities should be split by production stream:

- design and systems
- gameplay implementation
- UI and UX
- item and icon art
- world environment and effects
- combat feel and juice

Even if one person handles multiple roles, planning should still track work this way.

## 8. Documentation workflow

Keep three living design documents:

- product definition
- content backlog
- decision log

Do not bury current decisions inside long brainstorm files.

## 9. Prototype content gate

Only add content if it does at least one of these:

- proves a core mechanic
- reveals a balancing problem
- improves readability
- expands a real strategy branch

Do not add content only because the theme sounds cool.
