# 战斗空间与储物空间规格

## 1. 文档目的

这份文档用于定义首版中玩家拥有的两类主要物品空间：

- 战斗空间
- 储物空间

目标是把空间管理明确到足以支撑：

- UI 实现
- 道具摆放规则
- 职业交互
- 奖励与适应流程

## 2. 双空间核心规则

玩家拥有两个含义明确不同的空间。

### 战斗空间

战斗空间是主动空间。

只有放在战斗空间中的物品才会：

- 参与战斗
- 提供有效支撑
- 产生面向战斗的供能或逻辑收益

### 储物空间

储物空间是保留空间。

放在储物空间中的物品：

- 不会直接影响战斗
- 会被保留用于后续调整
- 用于支撑战斗之间的 build 适应

玩家应始终能清楚知道哪些物品是激活中的，哪些只是被保管。

## 3. 首版尺寸建议

### 战斗空间

建议初始大小：

- `6 列 x 5 行`
- 总计 `30 格`

原因：

- 足以支撑有身份感的 build
- 又不会大到让取舍失效
- 能兼容当前首批道具里 `1x1`、`1x2`、`2x1`、`2x2`、`L3`、`line3` 的组合

### 储物空间

建议初始大小：

- `4 列 x 4 行`
- 总计 `16 格`

原因：

- 足以放下转型件和备用件
- 又不足以让玩家什么都屯着
- 会迫使玩家决定哪些东西值得留着以后用

## 4. 放置规则

### 战斗空间放置

规则：

- 道具不能重叠
- 道具必须完整落在有效网格内
- 旋转后的道具也必须合法贴合
- 非法放置在松手前就应有明显预览

### 储物空间放置

规则：

- 储物空间同样服从形状与占位规则
- 储物空间是有格子的空间，不是抽象列表
- 玩家应感受到“保留物品”本身也有真实空间成本

这一点很重要，因为储物空间是策略层，不只是库存溢出区。

## 5. 空间交互规则

玩家应可以：

- 把物品从战斗空间拖到储物空间
- 把物品从储物空间拖到战斗空间
- 在允许时旋转物品后再摆放
- 在准备阶段整理两个空间

玩家不应可以：

- 通过放在储物空间里获得战斗收益
- 把道具塞进“抽象背包槽”从而绕开占位规则

## 6. 允许整理的时机

### 战前

玩家可以：

- 完整整理战斗空间
- 在战斗空间与储物空间之间调换物品
- 购买并放置新道具
- 把部分物品撤出当前激活布局

### 战后

玩家可以：

- 做有限但有意义的调整
- 从储物空间重新上场部分道具
- 根据奖励结果重构空间

设计规则：

- 玩家在每场战斗后都应觉得自己还能适应
- 系统不应让每一次都变成极重的“全量重摆”负担

## 7. 战后有限调整规则

建议的首版做法：

- 每场战斗后，玩家都可在准备阶段自由整理
- 但 UI 流程上应把它包装成一个短调整窗口，而不是巨大管理模式

这样可以兼顾两点：

- 适应仍然有乐趣
- 节奏仍然清楚

## 8. 扩容规则

首版应谨慎对待空间扩张。

建议规则：

- 战斗空间扩容应稀有且有分量
- 储物空间扩容可以略微常见一些
- 前期成长不应太快消解空间压力

原因：

- 空间压力本身就是项目身份的一部分
- 扩太多会削弱构筑取舍

## 9. DNA 驱动的职业空间变化

当玩家通过 DNA 解锁职业能力后：

- 战斗空间允许根据职业身份发生变化
- 这种变化应同时是可见的、且在机制上有意义
- 这种变化不能把核心空间压力完全抹掉

建议的首版理解方式：

- 职业变化可以改变战斗空间形状、特殊格，或结构倾向
- 职业变化应制造不同的摆放激励
- 职业变化应比单纯的隐藏数值更容易被玩家读懂

可接受的首版变化例子：

- 修士：战斗空间更接近阵图感或环形结构感
- 魔法师：战斗空间出现更明确的施法通道或奥术锚点格
- 人工智能：战斗空间出现结构化逻辑通道或处理区

控制规则：

- 职业空间变化必须有意义，但不能大到让每个职业都像在玩完全不同的棋盘游戏

## 10. 职业交互指引

职业可以影响空间，但必须保持克制。

适合首版的职业空间交互：

- 提高特定摆法的收益
- 提高特定形状道具的利用效率
- 给少量储物效率或结构化支撑收益

首版应避免这些职业交互：

- 大幅扩大棋盘大小
- 直接移除空间约束
- 因为原始容量更大而让某个职业显著强于其他职业

## 11. UX 要求

UI 必须清楚展示：

- 战斗空间边界
- 储物空间边界
- 激活与存放状态
- 非法放置状态
- 放置预览
- 在有需要时的旋转状态

UI 还必须让玩家明确知道自己当前是在：

- 准备阶段
- 即将进入战斗
- 查看战后奖励

UI 还必须通过以下方式清楚展示职业化空间变化：

- 棋盘边框变化
- 职业标记
- 若使用特殊格，则高亮这些特殊格
- 第一次解锁时展示变化前后对比

## 12. 验证问题

只有当以下问题大多能回答“是”时，这套规格才算成立：

- 玩家不经解释就能理解战斗空间和储物空间的差异
- 两个空间里的空间压力都是真实存在的
- 储物空间支持的是适应，而不是纯囤积
- 战斗空间既足够让人发挥创意，又足够逼出取舍
- 道具形状带来的是有趣摩擦，而不是纯粹麻烦
- 职业化棋盘变化是否让人兴奋，同时又不会看不懂

## 13. 对制作的直接影响

下一层规划应继续定义：

- 具体拖拽交互流程
- 放置高亮规则
- 储物空间扩容来源
- 战斗空间扩容来源
- 双空间界面的首版线框图
- 职业专属战斗空间变体
