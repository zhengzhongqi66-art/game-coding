# 经济与奖励模型

## 1. 文档目的

这份文档用于定义奖励、商店、reroll 与 run 内经济应如何服务于核心玩法循环：

- 在有限空间内构筑
- 进入战斗验证
- 在约束下做适应
- 再往前推进一个节点

经济系统不应只负责堆数值。
它应该持续制造“必须做选择”的压力。

## 2. 经济设计目标

奖励模型应稳定地产生以下感受：

- “我有选择，但空间不够全都拿。”
- “这个奖励很强，但可能逼我重构布局。”
- “我可以现在先稳住，也可以赌一个更强的引擎。”
- “这一局正在长成它自己的构筑故事。”

如果奖励只是让数值变大，却不改变结构，那么这个系统就是低效的。

## 3. 核心经济层

### A. Run 内主货币

首个制作版本建议只使用一种主要 run 内主货币：`精魄`。

定义：

- 精魄是跨世界通用货币
- 它代表可被提取与兑换的能力价值，而不是某个单独世界的本地货币
- 它既可以在 run 内消费，也可以在主神空间中承担部分长期兑换职能

这套货币应覆盖：

- 商店购买
- reroll
- 特定事件选择
- 如果后续加入，也包括修复或恢复类操作
- 在 Hub 或主神空间中兑换部分长期能力时的消耗

原因：

- 对早期玩家更易读
- 平衡复杂度更低
- 风险与回报的取舍更干净

在第一个 vertical slice 阶段，不建议同时引入多种 run 内主货币，除非单货币方案已被证明明显失效。

### B. 经验

经验应作为一条独立于精魄的成长信号存在于 run 过程中。

目的：

- 给 run 节奏提供稳定成长反馈
- 即便玩家跳过道具，也能获得基础奖励感
- 为后续等级制或阶段式成长预留空间

建议规则：

- 经验主要来自战斗结算
- 经验不能替代构筑决策本身
- 它应强化一局 run 的成长弧线，而不是压过构筑乐趣

### C. 奖励对象

奖励应被控制在少量、清晰的类别中：

- 经验
- 精魄
- 宝箱
- 名牌道具三选一
- 新道具选择
- 升级或精炼动作
- 工具性动作，例如移除、辅助旋转、后续可扩展的空间类操作
- 世界特有奖励

建议理解方式：

- 经验是基础成长奖励
- 精魄是最灵活的通用经济奖励
- 宝箱是较高波动的容器型奖励
- 名牌道具三选一是最直接的策略分支奖励

大多数情况下，玩家应是在“类别之间做选择”，而不是被动拿到一份平铺直叙的奖励。

### D. 元进度

元进度必须与 run 内经济分离。

前期元进度只应用于：

- 解锁道具家族
- 解锁世界变体
- 解锁商店可能性
- 解锁挑战修饰项

避免：

- 永久性数值膨胀
- 用元进度掩盖 run 设计本身不够好玩

## 4. 一局 run 内的奖励节奏

### 前期

前期 run 应优先服务于：

- 构筑身份成型
- 基础支撑件获取
- 第一套引擎的发现

建议的前期奖励表现：

- 道具选择多于纯货币
- 第一次 reroll 惩罚较低
- 更强调清晰度，而不是高随机性

### 中期

中期 run 应优先服务于：

- 结构性适应
- 针对性应对
- 协同深化

建议的中期奖励表现：

- 更强的分支化奖励
- 更昂贵的 reroll
- 更高的世界偏向奖励
- 更大压力去替换或移除弱件

### 后期

后期 run 应优先服务于：

- 已有投入的兑现
- boss 准备
- 有限但真实的补救窗口

建议的后期奖励表现：

- 更少的通用奖励
- 更高价值但风险更大的选择
- 更偏向 boss 应对的适应机会

## 5. 奖励来源设计

### A. 普通遭遇

普通遭遇应主要提供：

- 精魄
- 经验
- 小概率获得对方持有物品

建议体验目标：

- 稳定推进
- 低负担适应
- 偶尔通过缴获敌人物品带来惊喜

目的：

- 保持 run 节奏前进
- 鼓励渐进式塑形

### B. 精英遭遇

精英遭遇应提供：

- 更多精魄
- 更多经验
- 中概率获得对方持有物品
- 比普通战更强的构筑分支压力

目的：

- 制造有意识的紧张感
- 让冒险真正带来构筑方向改变

### C. Boss 遭遇

Boss 奖励应提供：

- 常规 Boss 级别的精魄与经验奖励
- 一次 `明牌道具三选一`

建议表现方式：

- 三个选项全部明牌展示
- 玩家是在做一次明确的策略承诺
- 这份奖励应比普通宝箱更“被设计过”、更有记忆点

当前备注：

- 职业技能解锁材料与兑换规则应独立定义
- 在后续正式确认前，不要默认把 Boss 奖励和 DNA 片段掉落绑定

Boss 奖励不应只是“更大的普通宝箱”。

### D. 事件

事件应更多考验判断，而不是纯战斗强度。

好的事件奖励可以包括：

- 用生命值换经济收益
- 破坏一个道具来换定向奖励
- 复制某类窄用途道具，但附带代价
- 锁入一个世界修饰项以换更高收益

## 6. 商店设计

### 首版商店规则

首版商店应保持小而易懂。

建议结构：

- 4 到 6 个出售位
- 1 个 reroll 按钮
- 1 个可选的移除或精炼服务
- 如有需要，保留 1 个明显的世界偏向位

开放规则：

- 每场战斗后的准备阶段，玩家都可以进入商店
- 玩家应稳定形成“战后准备阶段包含商店”的预期
- 商店出现频率高，但购买仍受精魄收入约束
- 地图上的特定节点还可以提供强化版商店入口

### 商店设计目标

商店应帮助玩家回答：

- 我能不能先稳住？
- 我能不能临时转型？
- 我能不能为了更强引擎再贪一把？

如果商店只卖泛用升级，那它在策略层面的工作是不够的。

## 7. Reroll 模型

Reroll 应该是有分量的经济决策，而不是条件反射。

建议的首版模型：

- 每次进商店的第一次 reroll 很便宜
- 同一次访问中的后续 reroll 逐次涨价
- 到下一家商店时价格重置
- 所有 reroll 都消耗精魄

原因：

- 允许玩家从一次糟糕商店中自救
- 防止无脑无限捞牌
- 在稳定性与贪心之间制造可见张力

失败信号：

- 玩家几乎每次都 reroll
- 玩家几乎从不 reroll，因为价格显然不合理
- reroll 的价值高于学习如何围绕现有资源适应

## 8. 构筑适应工具

奖励经济里应提供有限但关键的“改造 run”能力。

建议首版包含：

- 移除一个道具
- 升级一个道具
- 把一次奖励选择转成货币
- 如有需要，在重伤后给一次条件式修复机会

这些工具重要，是因为这个游戏的乐趣在于“聪明地重构”，而不只是不断往上叠东西。

## 9. 道具池的策略角色分布

奖励系统应跟踪的是“策略角色”，而不只是道具数量。

建议首批道具池按这些角色桶来控制：

- 引擎件
- 能源件
- 支撑件
- 转换件
- 防御件
- 经济或工具件
- 高风险高收益件

验证规则：

- 一次成功 run 通常应同时包含多个角色桶
- 奖励不应只大量产出收益件

## 10. 补救与防止过早判死刑

除非玩家连续做出错误决策，否则一局 run 不应过早显得“已经没救了”。

建议的防暴毙规则：

- 中期至少给一次清理弱布局的机会
- 至少保留一个稳定来源，能拿到基础支撑工具
- 避免奖励池完全阻断基本引擎的成型

这不意味着每局都要容易。
它意味着要保护“我仍然有机会适应”的感觉。

## 11. 首轮原型需要调的数字

团队应明确联动调这些参数：

- 每场遭遇的平均精魄产出
- 每场遭遇的平均经验产出
- boss 前平均能看到多少次道具选择
- reroll 价格曲线
- 移除或精炼服务价格
- 稀有奖励出现率
- 世界偏向奖励出现率
- 不同遭遇层级的敌人物品掉落概率
- 明牌名牌道具的质量与出现节奏

这些参数应一起看，而不是一个个孤立调整。

## 12. 验证问题

只有当团队对以下大多数问题都能回答“是”时，经济系统才算有效：

- 奖励是否经常迫使玩家做真实的布局调整？
- 玩家是否会因为结构代价而放弃看起来很强的奖励？
- 商店是否会制造转型时刻，而不是填充暂停？
- reroll 是否有用，但又不是强制动作？
- 玩家是否能解释自己为什么花费或保留精魄？
- 失败是否有时能追溯到经济决策，而不只是战斗数值？
- Boss 的明牌三选一是否能形成有记忆点的策略承诺？

## 13. 对制作的直接影响

这套规划后续应继续补充：

- 奖励表结构
- 商店库存逻辑
- reroll 价格缩放表
- 用于平衡跟踪的道具角色标签
- 经济遥测定义
- 职业技能解锁资源设计
- 战后准备阶段流程规格

如果这些长期模糊不清，构筑多样性会越来越难调。
