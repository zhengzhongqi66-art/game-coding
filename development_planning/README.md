Current project definition:

> A small-team indie game built around space-based build construction, auto-battle validation, looped dungeon worlds, and light meta progression.

Core interpretation for planning:

- The core is not "xianxia backpack" as a fixed theme.
- The core is "arrange a build in limited space, enter a themed world, validate it in combat, return to hub, adapt, and go again."
- Xianxia is the strongest first-world and first-build direction, not necessarily the only long-term umbrella.

- Keep speculative ideas outside these files until they are approved.
- When a direction changes, update `01_product_definition.md` and `06_risks_and_decisions.md` first.

Bilingual documentation rule:

- Future planning files should be created in both English and Chinese.
- Use the base filename for English and the `_cn` suffix for Chinese.
- If one version changes, the paired version should be updated in the same round.
