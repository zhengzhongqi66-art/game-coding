# Roadmap And Milestones

## 1. Production goal

The immediate goal is not full release.
The goal is to reach a convincing playable vertical slice that proves the project's core loop.

## 2. Milestone breakdown

### Milestone 0: Pre-production lock

Target outcome:

- one clear product statement
- one chosen engine
- one chosen first world
- first module map
- first content boundary

Deliverables:

- planning docs approved
- wireframe of hub and construction screen
- item schema draft
- combat phase draft

### Milestone 1: Core interaction prototype

Target outcome:

- arranging items in limited space is already satisfying

Deliverables:

- placement grid
- rotate and move support
- item size handling
- adjacency detection
- direction preview
- simple activation preview

Success check:

- moving one tile frequently changes projected value
- testers voluntarily rearrange for improvement instead of stopping at first fit

### Milestone 2: Combat validation prototype

Target outcome:

- a placed build can enter battle and resolve clearly

Deliverables:

- battle scene
- simple enemy wave
- action sequencing
- damage and shield resolution
- combat log or event feed
- win and loss summary

Success check:

- testers can explain why they won or lost
- testers can identify the main trigger chain or failed support link

### Milestone 3: Full run loop prototype

Target outcome:

- hub -> encounter -> reward -> rebuild -> next encounter is functional

Deliverables:

- hub flow
- encounter map or node path
- reward screen
- shop or draft choice
- one elite
- one boss

Success check:

- a complete short run feels coherent
- reward choices regularly cause meaningful rebuild decisions

### Milestone 4: First-world vertical slice

Target outcome:

- the first world has a real identity

Deliverables:

- xianxia world art pass
- first stable item pool
- five-element placement rules
- energy support pieces
- first event nodes
- first boss identity

Success check:

- the game has a clear screenshot and trailer hook
- the world rule measurably changes how players construct builds

### Milestone 5: Demo candidate

Target outcome:

- external players can understand and enjoy the loop

Deliverables:

- onboarding
- basic balance pass
- minimal meta unlocks
- save support
- bug triage
- performance sanity pass

Success check:

- outside testers understand the core loop in one session
- outside testers can explain at least one loss and one next-run adjustment

## 3. Suggested timeline for a small team

This is an aggressive but realistic target if the team stays focused.

- Week 1 to 2: pre-production lock
- Week 3 to 4: core interaction prototype
- Week 5 to 6: combat validation prototype
- Week 7 to 9: full run loop prototype
- Week 10 to 12: first-world vertical slice
- Week 13 to 16: demo polish and testing

If team capacity is lower, extend the timeline rather than expanding scope.

## 4. Scope priorities

If time becomes tight, protect these in order:

1. Space arrangement depth
2. Readable combat payoff
3. One clean run loop
4. One memorable world identity
5. Minimal progression

Cut these first if needed:

- extra worlds
- extra characters
- PvP features
- heavy narrative sequences
- cosmetic side systems

## 5. Demo content recommendation

Recommended demo contents:

- 1 hub
- 1 world
- 1 short route
- 1 boss
- 20 to 30 curated items
- 10 to 15 minutes for a first run

That is enough to prove the product without pretending to be the final game.
