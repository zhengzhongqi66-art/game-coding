# 玩法结构详解：空间构筑 + 轮回副本 + 内容取样

## 这份文档解决什么问题

前面的文档已经把方向收敛出来了，但还有一个问题：

- 玩法说得还不够“像真的游戏”
- 《无限恐怖》式结构虽然提到了，但没有明确说清楚它到底解决了什么问题
- “去不同作品/题材里取样”的方法，也还没有变成可执行的设计语言

所以这份文档重点回答：

1. 这个游戏实际玩起来一轮到底是什么样
2. 《无限恐怖》式结构为什么适合我们
3. “剧本取样 / 世界取样”到底怎么做，才不会变成东拼西凑

---

## 一、先把游戏讲成人能一下听懂的话

### 当前最接近的项目定义

**一个带轻度无限流结构的空间构筑独立游戏。**

玩家会反复经历这个循环：

1. 在中枢空间整理自己的构筑空间
2. 选择进入一个副本世界
3. 在副本里战斗、拿奖励、触发事件或支线
4. 回到中枢空间，用这次奖励继续强化构筑
5. 进入下一轮，打更深的内容

所以这个游戏不是单纯“摆包然后打一架”，而是：

**空间构筑 -> 进副本验证 -> 回中枢再调整 -> 再进副本**

这个来回往复，就是整个项目的真正骨架。

---

## 二、玩法到底是什么，不是什么

## 1. 玩法核心不是“背包”

“背包”只是最容易理解的形式，但它不是玩法本质。

玩法本质是：

**在一个有限空间里安排 build，让空间结构直接影响战斗结果。**

所以这个空间可以叫：

- 背包
- 储物袋
- 纳戒
- 阵图
- 轮回匣
- 灵台

但无论叫什么，真正重要的是：

- 格子有限
- 大小不同
- 摆放有讲究
- 邻接会联动
- 朝向会影响输出
- 某些物品负责供能
- 某些物品负责吃条件爆发

## 2. 玩法核心也不只是“自动战斗”

自动战斗只是结算方式，不是核心趣味本身。

真正的趣味来自三个连续动作：

### 第一层：排布

玩家思考的是：

- 这个东西摆哪
- 它应该靠谁
- 它应不应该转个方向
- 这个大件值不值得占这么多空间

### 第二层：组合

玩家真正兴奋的是：

- 这两个东西本来没想到能一起用
- 换一格居然整套 build 活了
- 供能件终于把核心法器喂起来了

### 第三层：开奖

自动战斗最重要的作用是：

**把玩家刚才的空间决策“开奖”给他看。**

也就是说，自动战斗本质上是一个“验证 build 是否成立的可视化结算器”。

---

## 三、一局游戏玩起来是什么感觉

这里我们把它讲成非常具体的一局。

## 1. 局外：回到中枢空间

你打完上一个副本，回到一个类似“轮回中转站”的地方。

在这里你会做几件事：

- 看本轮拿到了什么奖励
- 买新物品 / 替换旧物品
- 调整自己的空间构筑
- 选下一轮去哪个世界
- 决定要不要赌一个支线风险

这个地方类似《无限恐怖》里的“回主神空间”。

它的作用不是讲故事，而是给玩家一个很清楚的准备阶段：

**上一轮我发生了什么 -> 这一轮我怎么变强 -> 下一轮我去哪**

## 2. 局内：进入副本世界

进入副本后，当前玩法应该偏轻量，不做大地图冒险，先做：

- 小地图关卡
- 连续战斗节点
- 事件节点
- 奖励节点
- 精英/Boss 节点

也就是说，副本世界不是开放探索，而更像：

**一个带主题规则的战斗试炼场**

## 3. 战斗

战斗本身主要靠自动结算。

你进入战斗前已经决定好了：

- 空间怎么摆
- 哪些物品作为主输出
- 哪些物品负责供能
- 哪些联动负责爆发

战斗里最关键的不是操作，而是：

- 你能不能看懂 build 在运转
- 你能不能看到它为什么赢
- 你能不能看出自己下一把想怎么改

## 4. 回中枢

副本结束后，回中枢空间：

- 拿奖励
- 换 build
- 再选世界

这就形成一个很强的循环感：

**构筑不是一次性的，而是会随着你去不同副本而不断变化。**

## 四、《无限恐怖》式结构为什么对我们特别有价值

你提到《无限恐怖》，这个参考非常关键，因为它帮我们同时解决了 4 个问题。

## 1. 它解决“战斗理由”问题

很多构筑游戏都有一个很普通的问题：

- 为什么我又打一关？
- 为什么换场景了？
- 为什么敌人类型突然变了？

如果只是普通 roguelite，这些变化可以接受，但不够“顺”。

而无限流/主神空间结构天然就能解释：

- 你为什么不断进入新世界
- 你为什么不断战斗
- 你为什么打完又回中枢
- 你为什么能获得奖励并兑换能力

也就是说：

**它让“不断刷副本”这件事不再只是游戏流程，而变成世界观逻辑。**

## 2. 它解决“题材扩展”问题

如果项目被锁死在单一修仙世界，问题是：

- 敌人类型会有限
- 场景变化会有限
- 美术风格比较容易单一
- 道具体系也容易受限制

而无限流结构允许：

- 修仙秘境
- 都市怪谈
- 克系古堡
- 废土实验室
- 校园诡境
- 古代战场

这些世界都共存。

这样一来，内容来源会自然很多。

## 3. 它解决“素材来源”问题

这是你提到最重要的一点。

很多小团队自己做原创世界，最难的不是玩法，而是：

- 世界设定要想很久
- 敌人理由要想很久
- 场景和关卡为什么存在，也要想很久

无限流很聪明的地方在于：

它不是要求你从 0 造一个完整大世界，
而是允许你从“大家已经熟悉的题材模板”里抓取高辨识元素，再统一纳入一个轮回结构。

这就极大降低了内容设计难度。

## 4. 它解决“轻叙事承载”问题

你们又不想做纯剧情游戏，但也不想完全没有目标。

无限流结构刚好适合“轻叙事”：

- 中枢空间有主线
- 不同副本有主题
- 某些副本能出支线
- 敌对构筑者 / 平行时空队伍能作为故事钩子

这样叙事不会压过玩法，但世界也不会太空。

## 五、什么叫“剧本取样 / 世界取样”

这里很容易误解。

“取样”不是让我们直接照搬某部作品的剧情，而是：

**从成熟题材中提取“高识别度规则单元”，再转译成玩法和美术资产。**

这个很重要。

## 1. 不取剧情，取“题材规则”

比如一个“校园诡境”世界，我们不该直接想：

- 剧情要不要像某部校园恐怖片

而应该先想：

- 校园世界最典型的空间是什么
- 敌人行为规则是什么
- 哪些视觉元素一眼就能认出题材
- 它最适合给构筑系统带来什么玩法变化

也就是说，不取故事线，优先取：

- 空间
- 规则
- 敌人行为
- 视觉符号
- 节奏气质

## 2. 不取设定百科，取“可玩模板”

举例：

### 修仙世界

可取样内容：

- 五行
- 法器
- 丹药
- 阵法
- 灵气供能

转成玩法后就是：

- 相生相克
- 供能件
- 爆发件
- 范围件
- 朝向件

### 都市怪谈世界

可取样内容：

- 电梯
- 便利店
- 夜路
- 监控
- 规则怪谈

转成玩法后就是：

- 视野限制
- 延迟触发
- 条件判定
- 异常状态
- 特殊敌人机制

### 克系古堡世界

可取样内容：

- 污染
- 理智
- 异形肉块
- 仪式
- 召唤

转成玩法后就是：

- 污染格子
- 侵蚀 build
- 召唤物
- 周期性场地变化

## 3. 取样的最小单位

为了避免项目散掉，我建议每个世界只取这 5 类东西：

1. `3 个视觉符号`
2. `2 个敌人机制`
3. `1 个场地规则`
4. `1 套主题奖励`
5. `1 个支线事件钩子`

这样每个世界就够用了，不会膨胀成一整套重内容。

## 六、如何把“世界取样”落成真实玩法

这里我们给一个完整例子。

## 例子：修仙秘境世界

### 视觉符号

- 悬浮山门
- 炼丹炉
- 飞剑残阵

### 敌人机制

- 剑修傀儡：前方直线高伤
- 炼气妖藤：会吸收木系效果

### 场地规则

- 每隔几秒会产生灵气脉冲，给某些格子充能

### 奖励取样

- 法器
- 丹药
- 灵石

### 支线钩子

- 破损的阵眼要不要修
- 赌一把修复后拿大奖，还是稳妥跳过

这一整套出来以后，玩家感受到的是：

“这是修仙世界”

但设计师真正做的事情，其实是：

“做了一个带木/火/灵气充能特征的战斗关卡包”

这就是取样方法的价值。

## 七、项目最值得保留的玩法主轴

基于目前所有讨论，项目主轴已经比较清楚了。

## 主轴 1：空间构筑

这是项目根，不讨论。

玩家一定要一直感到：

- 一格很重要
- 位置很重要
- 顺序很重要
- build 结构比数值更重要

## 主轴 2：副本轮回

这个是项目骨架。

它提供：

- 目标感
- 进度感
- 世界切换的理由
- 轻叙事容器

## 主轴 3：主题规则副本

每个世界不靠大量文本立住，而靠：

- 主题规则
- 场地变化
- 奖励风格

## 主轴 4：轻 meta

这个要有，但很轻。

玩家回中枢空间能做的事情应该是：

- 购买新物品
- 解锁新规则
- 扩容一点点
- 选下一站风险

而不是做大量养成和剧情阅读。

---

## 八、当前最合理的首版边界

如果按你现在的想法继续收敛，首版最合理的边界是：

## 中枢空间

- 1 个

## 主玩法载体

- 1 套有限空间构筑系统

## 世界数量

- 2 个

建议：

- `1 个主世界：修仙秘境`
- `1 个差异世界：都市怪谈 / 校园诡境`

## 对抗形式

- 以 `PVE` 为主
- 加入少量 `镜像 build 敌人`

## 成长

- 轻成长
- 轻解锁
- 不做复杂天赋树

---

## 九、和美术伙伴最值得对齐的部分

如果这份文档是拿去和美术伙伴讨论，我建议重点聊这 6 件事：

1. 项目更适合纯修仙统一美术，还是“中枢空间 + 多世界题材”结构
2. 空间载体更适合叫背包、纳戒、阵图、轮回匣里的哪种
3. 如果采用取样法，首版两个世界怎么搭配视觉最稳
4. 轻松有梗的气质，主要落在 UI、敌人、命名还是结算表现
5. 如果修仙只是一个分支，那么整个项目的视觉主母题到底是什么
6. 哪种表达方式最能和 Backpack 类对标形成距离

---

## 十、现在可以暂定的项目描述

为了方便继续讨论，目前最适合的暂定描述是：

**一个带轻度无限流结构的空间构筑独立游戏。玩家在中枢空间整理 build，进入不同主题副本战斗与取样，再回到中枢继续强化。修仙可以是首个最稳的世界和 build 分支，但项目整体不必被修仙锁死。**

---

## 十一、最后的关键判断

目前最值得保留的不是“修仙背包乱斗”这几个字，而是下面这个结构：

**有限空间构筑 + 自动战斗 + 中枢回合 + 世界取样 + 轻 meta**

这才是项目真正有潜力的地方。

修仙、纳戒、主神空间、怪谈副本，这些都是围绕这个结构服务的。
