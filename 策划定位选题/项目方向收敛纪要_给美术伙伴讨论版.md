# 项目方向收敛纪要

## 目的

这份文档用于和美术伙伴讨论项目方向，不是最终立项书，也不是详细设计稿。

目标是把我们目前已经讨论出的：

- 核心结论
- 机制判断
- 对标理解
- 玩法方向
- 项目边界
- 暂不确定项

统一收敛成一个清晰版本，方便一起讨论、取舍和补充。

---

## 一、当前最重要的结论

### 1. 项目不一定要锁死在“修仙”

目前更合理的理解，不是做一个纯修仙题材游戏，而是：

**做一个“空间构筑 + 自动战斗 + 轮回副本 + 轻 meta”的独立游戏。**

其中：

- `修仙` 可以是其中一个 build 分支
- `背包` 也不一定是唯一载体
- 真正核心是：**有限空间里的 build 排布**

也就是说，修仙现在更像一个很好的流派和视觉方向，但不一定要成为整个项目唯一的母题。

### 2. 项目核心不是文本，而是系统乐趣

我们已经明确不想把项目做成：

- 靠大量文本支撑趣味
- 靠长剧情推动玩家
- 靠网剧感题材出圈

我们更想做的是：

- 规则清楚
- 空间安排有趣
- build 联动有惊喜
- 自动战斗有爆发感
- 玩家会因为“摆法”和“组合”想再来一局

### 3. 项目气质希望偏轻松有梗

但“梗”不是要靠大量文案去撑，而更适合体现在：

- 道具命名
- 物品联动效果
- 世界主题反差
- 结算语气
- 敌人和副本机制表现

一句话就是：

**梗是调味，不是主菜。**

### 4. 项目现在更适合做小圈层中文甜品型独游

暂时不以大规模市场为目标，不先考虑全球化表达，而是先以：

- 中文区
- 小圈层独游玩家
- 觉得“这玩意儿有点新、有点有趣”的人

为首批目标用户。

## 二、当前项目的母结构

## 1. 核心玩法层

玩家要在一个有限空间里安排 build。

这个空间的形式可以是：

- 背包
- 储物袋
- 纳戒
- 阵图
- 轮回匣
- 主神空间里的个人构筑格

但无论换什么皮，核心都不变：

**空间排布决定战斗表现。**

## 2. 战斗层

战斗目前倾向：

- 自动战斗
- 战前构筑，战中观看结算
- 爽点来自 build 爆发，而不是高强度操作

战斗理由希望更丰富，而不只是“打一关”：

- 进入副本世界
- 战斗/探索/触发支线
- 回到中枢空间
- 用奖励强化 build
- 进入下一轮

## 3. 中枢空间层

目前讨论出的想法很接近“无限流 / 主神空间 / 轮回空间”结构：

- 每一轮进入不同副本世界
- 每轮结束回到中枢空间
- 获得奖励、购买能力、触发支线
- 继续下一轮

这个结构的好处非常大：

- 战斗理由天然成立
- 世界素材来源很丰富
- 美术题材可以扩展
- 道具、敌人、世界观不容易快速枯竭

## 4. 对抗层

正常推进以 `PVE` 为主。

`PVP` 不建议一开始就做联网，而更适合做：

- 异步 PvP
- 平行时空 build 残影
- 历史最强自己
- 预设镜像对手

这样既能有对抗感，又不至于把开发复杂度拉爆。

---

## 三、当前明确成立的玩法乐趣

经过前面的讨论，现在成立的乐趣不是单一的，而是组合型：

## 1. 空间构筑乐趣

玩家整理空间本身就有乐趣：

- 大件物品难摆但很强
- 小件物品适合补缝
- 朝向、邻接、占位都很重要
- 一格之差可能整套 build 都变

这是整个项目最底层的 fun。

## 2. 自动战斗爆发乐趣

战斗不是主操作，而是对构筑结果的“开奖”：

- build 转起来很爽
- 联动打出来有观赏性
- 玩家能看到“为什么赢了”

## 3. 意想不到的物品组合

项目很重要的一点是：

**玩家应该不断发现本来没想到会一起工作的东西，居然真的配起来了。**

这意味着设计重点要放在：

- 邻接
- 联动
- 条件触发
- 反常识组合

而不是单纯堆数字。

## 4. 轮回推进的目标感

因为有中枢空间和副本结构，玩家的动力就不只是“再打一关”，而是：

- 这轮拿什么奖励
- 要不要赌支线
- 回去买什么能力
- 下轮会碰到什么世界

这会让项目比纯构筑小游戏更有“往前走”的感觉。

---

## 四、当前最值得保留的机制关键词

以下这些关键词，基本可以视为项目的核心候选资产：

- 空间构筑
- 自动战斗
- build 爆发
- 邻接规则
- 朝向规则
- 充能/供能
- 意外组合
- 副本轮回
- 中枢空间
- 轻 meta
- 异步镜像对手
- 轻松有梗

## 五、我们对“修仙背包乱斗”的最新判断

## 1. 它仍然是非常好的方向

`修仙背包乱斗` 依然是目前非常强的候选方向，因为它有这些优点：

- 中文区理解成本低
- 道具和法器天然适合做 build
- 五行规则很适合进入空间安排逻辑
- 美术上有很强的发挥空间
- 一句话就比较好讲明白

## 2. 但它不一定是最终母题

当前更合理的判断是：

- 修仙可以作为第一个最好落地的流派
- 但整个项目不一定要被“修仙”锁死
- 如果采用轮回空间结构，那么修仙可能只是第一个世界、第一类 build，或者一个核心分支

## 3. 如果保留修仙方向，最值得保留的差异化

如果我们继续保留修仙元素，那么最值得坚持的不是“仙侠皮肤”，而是这两层：

### 五行相生相克进入空间规则

不是战斗里单纯克制，而是摆包逻辑本身受影响：

- 水挨木有收益
- 木挨火能增强
- 相克会引发副作用

这样五行就不是设定，而是玩法。

### 法器朝向与灵气回路

比如：

- 飞剑吃朝向
- 丹炉吃包围
- 灵石负责供能
- 符箓吃连线

这会让我们的空间构筑语言更独特。

## 六、对标作品的核心结论

## 1. Backpack Hero

链接：

- [Steam 商店页](https://store.steampowered.com/app/1970580/Backpack_Hero/)
- [Destructoid 评测](https://www.destructoid.com/reviews/review-backpack-hero/)

我们对它的总结：

- 核心玩法非常强
- 背包空间规则设计很聪明
- 物品位置真的会改变战斗方式
- 但外围系统偏重，节奏被村庄和故事模式拖慢

**我们该学的：**

- 空间规则深度
- 物品的摆放人格
- 资源占位带来的取舍

**我们不该学的：**

- 首版就做太多外围系统
- 首版就把玩法做成“大而全冒险游戏”

## 2. Backpack Battles

链接：

- [Steam 商店页](https://store.steampowered.com/app/2427700/Backpack_Battles/)
- [PC Gamer 报道](https://www.pcgamer.com/games/arrange-your-items-to-optimize-your-fights-in-asynchronous-autobattler-backpack-battles/)

我们对它的总结：

- 更聚焦
- 更产品化
- 更像 Steam 爆款
- 核心循环被提纯得非常明确

**我们该学的：**

- 一句话玩法表达
- 主循环短而纯
- build 爆发的观感
- 少做无关外围系统

**我们不该照抄的：**

- 高强度 PvP 平衡负担
- 一开始就做异步对战为主

## 3. 当前总体判断

如果要给学习比例，我会建议：

- `70% 学 Backpack Battles`
- `30% 学 Backpack Hero`

因为我们现在更需要的是：

- 聚焦
- 容易出原型
- 少文本负担
- 容易做 Demo

## 七、之前筛选出的高优先级玩法方向

前面我们已经从大量选题里筛过一轮，最强的方向主要集中在这几个：

- 修仙背包乱斗
- 弹珠驱魔
- 县城停车神功
- 祭坛拼图防线
- 梗卡连锁店

这几类的共同特点是：

- 一句话讲得清楚
- 不依赖海量文本
- 10 分钟内能感知乐趣
- 很适合做 Steam Demo

## 其中，修仙/轮回空间构筑为什么仍然是当前最强候选

因为它能兼顾：

- 空间 build
- 自动战斗
- 世界容器
- 轻 meta
- 题材扩展性

相比之下：

- `弹珠驱魔` 更适合纯玩法 demo
- `县城停车神功` 更像很有特色的空间调度 puzzle
- `祭坛拼图防线` 更偏 tower defense 变体

而“轮回空间构筑”更像一个可持续延展的小型独游项目母结构。

## 八、当前项目边界

## 1. 已经明确不想走的路

- 不想做特别像网剧主题的东西
- 不想靠大量文本撑趣味
- 不想首版就做复杂剧情
- 不想系统层面过重
- 不想首版就大而全

## 2. 可以接受的内容

- 一点轻 meta
- 一点轻叙事
- 一点轻成长
- 有一个主线容器
- 有轻度异步镜像对抗

## 3. 当前最合理的复杂度判断

可以做：

- 一个中枢空间
- 一个主玩法闭环
- 一套 build 系统
- 少量流派
- 少量永久解锁

先不要做：

- 多章节大叙事
- 多个完整世界同时上线
- 多角色系统
- 联机 PvP
- 复杂养成树

## 九、现在还没有定下来的关键问题

这部分是接下来和美术伙伴最值得讨论的。

## 1. 母题到底叫什么

目前更像下面两种方向之一：

- `修仙背包乱斗`
- `轮回空间构筑游戏`

这两个方向的区别在于：

- 前者更聚焦，更容易第一眼理解
- 后者更有延展性，更适合包容多个世界与流派

## 2. 空间载体叫什么

我们已经明确：

- 背包只是其中一种表达
- 也可能是：
  - 纳戒
  - 储物袋
  - 阵图
  - 轮回匣
  - 主神格

这会直接影响：

- UI 风格
- 美术视觉
- 世界观气质
- 和对标作品的距离

## 3. 首版副本世界做哪两个

如果做轮回空间结构，首版不能贪多。

建议只做：

- `1 个主副本`
- `1 个额外风格副本`

可选方向目前包括：

- 修仙秘境
- 都市怪谈
- 废土实验室
- 克系古堡
- 校园诡境

## 4. 修仙在项目里的地位

这是目前很关键但还没完全定的点：

- 修仙是整个项目母题？
- 修仙是第一个世界？
- 修仙是一个 build 分支？

这三个答案会直接影响后面的：

- 美术风格统一性
- 世界观写法
- 道具体系

---

## 十、目前最适合讨论的结论版本

如果现在要带着一个“暂定共识版本”和美术伙伴聊，我建议用下面这段：

### 暂定共识版本

我们现在倾向做的，不是单纯修仙游戏，也不是纯背包游戏，而是：

**一个偏轻松有梗的小体量独立游戏，核心是“有限空间里的 build 排布 + 自动战斗 + 轮回副本推进”。**

其中：

- 背包只是空间载体的候选之一
- 修仙是非常强的方向，但不一定是唯一母题
- 战斗以 PVE 为主，未来可加入异步镜像对抗
- 轻 meta、轻叙事、轻成长可以有，但不能压过核心玩法
- 项目第一目标不是讲大故事，而是先让“空间构筑 + build 爆发”足够好玩

---

## 十一、建议和美术伙伴重点聊的 5 个问题

1. 你更喜欢项目整体是“修仙统一风格”，还是“轮回空间多世界风格”？
2. 你更偏好“背包/储物袋/纳戒/阵图”哪种空间载体的视觉表达？
3. 如果只做一个主世界，修仙是否依然是最适合首版的世界？
4. 你觉得“轻松有梗”的表达，更适合放在 UI、道具、敌人，还是角色和结算？
5. 从美术成本和统一性上，首版是做纯修仙更稳，还是做“修仙 + 一个异世界副本”更合理？
